Smart Markdown — 帮助文档

Smart Markdown 是一款支持 Markdown 编辑、代码编辑、双向链接、SMD 单元格编辑、代码编译运行、本地评测、AI 助手、OpenJudge 集成等功能的全功能文本编辑器。

快速入门

1. 打开文件夹：点击「文件」菜单选择「打开目录」或通过快捷键，选择一个文件夹作为工作区根目录。文件树将显示该目录下的所有文本文件。

2. 打开文件：在左侧文件树中单击文件以临时预览标签页（斜体标题）打开，多次单击复用同一标签页；双击永久打开。编辑临时标签页内容后自动提升为永久标签页。

3. 编辑与保存：直接编辑文件内容，支持 Ctrl+S 保存。新建文件使用 Ctrl+N。

4. 预览 Markdown：打开 .md 文件后，点击工具栏「预览」按钮或 Ctrl+Shift+P 进入全屏渲染预览模式。支持分屏预览（Ctrl+P）。

5. 代码编辑与运行：打开 .cpp、.py 等代码文件，自动切换为代码编辑模式（语法高亮、行号、LSP 代码补全）。按 F5 编译运行。

6. 探索更多：左侧活动栏提供搜索、AI 助手（含错题本）、设置（位于底部）、评测（含 OpenJudge）等功能入口。右侧面板提供历史记录、大纲、标签、反链。

文件管理

文件树

左侧文件树以树状结构展示当前工作区根目录下的所有文件和文件夹。支持的操作包括：

  • 单击：以临时预览标签页打开文件（斜体标题），多次单击复用同一标签页
  • 双击：永久打开文件（或提升已有预览标签页）
  • 右键：弹出上下文菜单，支持新建文件（默认 .md）、新建文件夹、重命名、删除
  • 拖拽：在文件树内拖拽移动文件或文件夹，自动进行路径同步
  • Delete 键：选中文件/文件夹后直接删除（弹出确认提示）

文件树工具栏

文件树上方为工具栏，显示当前文件夹名称（路径过长时自动省略），右侧提供操作按钮：

  • 新建文件（Ctrl+N）：在文件树中内联创建新文件，立即进入重命名状态，确认后自动打开
  • 新建文件夹：在文件树中内联创建新文件夹
  • 一键收起：收起所有已展开的文件夹
  • 刷新：手动刷新文件树

面包屑路径

文件树顶部显示当前根目录的完整路径，每个路径段可点击快速跳转。路径过长时自动换行不撑宽左侧面板，根目录切换时同步更新。

文件操作

  • 新建文件（Ctrl+N）：在文件树中内联创建新文件，确认后自动永久打开
  • 保存（Ctrl+S）：保存当前编辑的文件
  • 另存为（Ctrl+Shift+S）：将当前文件另存为新文件，对话框自动定位到上次使用的文件夹
  • 关闭：关闭标签页时若文件有未保存的修改，会提示保存/不保存/取消

文件类型支持

支持 60+ 种常见文本文件格式，包括：.md、.txt、.c、.cpp、.py、.js、.html、.css、.json、.xml、.yaml、.rs、.go、.java、.sql 等。所有文件以纯文本形式呈现在编辑器中。

标签页管理

  • 预览标签页：单击文件以临时标签页（斜体标题）预览，多次单击复用；双击永久打开；编辑临时标签页内容后自动提升为永久标签页
  • 切换联动：切换标签页时，文件树自动选中对应的文件并展开折叠的父级目录
  • 等宽/非等宽：支持两种标签宽度模式（设置面板 → 外观）。等宽模式固定 140px，文件名左对齐溢出省略；非等宽模式标签宽度自适应文件名内容
  • 标签页高度：可在设置面板 → 外观中调节，范围 20–40px，默认 26px
  • 拖拽重排：拖拽标签页可重新排序，通过 DragOverlay 覆盖层实现浮动标签跟随鼠标，避免被关闭按钮遮挡
  • 切换目录：切换根目录时自动关闭临时预览标签页及不在新目录中的标签页（未保存文件提示确认）

编辑器

编辑器根据文件扩展名自动切换为合适的编辑模式，共支持六种模式：

Markdown 编辑模式（.md, .txt 等）

  • [[wikilink]] 自动补全：输入 [[ 时弹出文件名列表，方向键选择，Tab 补全并自动闭合 ]]
  • #tag 自动补全：输入 # 时弹出已有标签列表，Tab 补全
  • Markdown 语法高亮

代码编辑模式（.cpp, .py, .js 等）

提供 IDE 风格功能：

  • 语法高亮：支持 C/C++，Python。颜色跟随主题自动切换
  • LSP 语义高亮：C/C++ 使用 clangd，Python 使用 Jedi 提供函数名、参数名、变量名等语义级高亮，覆盖正则无法处理的场景
  • 行号显示：左侧行号区域，跟随字体缩放同步更新
  • 自动缩进：Enter 自动保持缩进，智能处理 { } 和 Python 冒号缩进
  • 括号补全：输入 { ( [ " ' 时自动插入匹配对
  • 智能退格：退格键以缩进步长为单位删除前导空白；空括号对中间退格同时删除左右括号
  • 行注释切换：Ctrl+/ 切换注释/取消注释
  • 缩进调整：Ctrl+[ 向左缩进，Ctrl+] 向右缩进
  • 代码补全：Ctrl+I 或输入 . :: -> 自动触发。C/C++ 使用 clangd LSP 后端，Python 使用 Jedi 后端
  • 悬停类型提示：鼠标悬停在变量/函数上 400ms 后，以自定义浮动弹窗显示类型信息，主题感知
  • 函数签名帮助：输入 ( 后 200ms 弹出函数签名浮动窗，显示活动参数高亮、文档和重载导航，始终位于文本光标上方
  • 诊断波浪线：红色错误 / 黄色警告波浪下划线，鼠标悬停显示诊断信息
  • 诊断面板：Ctrl+D 切换底部诊断面板（
  • 括号对着色：3 种颜色按嵌套层级循环（金色/紫红/浅蓝），未配对括号显示红色，支持跨行追踪
  • 指针/引用高亮：C++ 声明上下文中 * 和 & 运算符以关键字蓝色高亮（正则快速通道 + clangd 语义驱动双机制）
  • 当前行高亮：当前编辑行以特殊背景色高亮
  • 选中保留语法色：选中文本时保留原语法高亮颜色，仅叠加半透明选中背景
  • 搜索高亮：搜索结果在编辑器中高亮显示，主题切换时同步更新颜色
  • 大纲跳转高亮：从大纲面板跳转后目标行 1 秒自动消失的临时高亮

其他模式

  • PDF 阅读模式：打开 .pdf 文件时自动切换，使用 QPdfView + QPdfDocument 渲染
  • SMD 编辑模式：打开 .smd 文件时自动切换，单元格分块编辑（详见 SMD 编辑器章节）

字体缩放

支持对编辑器和 Markdown 预览进行字体缩放，范围 50%~300%：

  • 工具栏缩放按钮（− / 百分比 / + / 重置）
  • Ctrl+= 放大、Ctrl+- 缩小、Ctrl+0 重设为默认缩放比例
  • Ctrl+鼠标滚轮 缩放
  • 触控板缩放手势

Markdown 预览

打开 .md 文件时可切换至渲染预览模式，基于 WebEngine 渲染引擎。预览视图采用懒创建 + 释放策略，首次进入预览时创建，退出预览时释放 WebEngine 控件，避免无预览时占用 Chromium 进程内存。

全屏预览

点击工具栏「预览」按钮或按 Ctrl+Shift+P 进入全屏预览模式。支持以下渲染特性：

  • GitHub Flavored Markdown：标准 Markdown 语法（标题、列表、表格、代码块、引用等）
  • LaTeX 数学公式：$...$ 行内公式、$$...$$ 块级公式（基于 KaTeX），支持 \(...\) 和 \[...\] 备用定界符
  • Mermaid 图表：```mermaid 代码块渲染为流程图、时序图、甘特图等 SVG 图表
  • 代码块语法高亮：使用 C++ 端预处理方案，复用编辑器语法高亮规则，支持 C/C++ 和 Python
  • WikiLink：[[文件名]] 自动识别为超链接，点击可跳转至对应文件
  • Tag：#tag 自动识别为可点击标签
  • 代码块运行按钮：预览代码块右上角显示 ▶ 运行按钮，点击可编译运行

分屏预览

点击工具栏「分屏」按钮或按 Ctrl+P 进入分屏预览模式。编辑器区域被竖直分隔条分为左右两部分：左侧为 Markdown 源码，右侧为渲染预览。

  • 全屏预览与分屏预览互斥（开启一个自动关闭另一个）
  • 右侧预览采用防抖延迟更新策略（默认 500ms，可在设置面板中调节 100–2000ms）
  • 分屏比例可在设置面板中调节（30%~70%）
  • 切换文件时自动记忆各标签页的预览状态
  • 预览引擎与全屏预览共享同一套预处理管线（代码高亮、WikiLink、Tag 处理）

PDF 导出

工具栏「导出 PDF」按钮或 Ctrl+E 将当前 Markdown 文档导出为 PDF。使用临时隐藏 WebEngine 视图渲染页面（含 Mermaid 图表等异步内容），以浅色打印主题输出为 PDF 文件。

提示：非 .md 文件下预览按钮、分屏按钮和导出 PDF 按钮自动隐藏且快捷键无效。

SMD 编辑器

.smd 文件采用 ---smd:<type> 分隔符的单元格分块格式，类似 Jupyter Notebook 的交互模式。支持 Ctrl+T 在 .md ↔ .smd 之间双向转换，保留内容映射关系。

单元格类型

  • Markdown 单元格：Markdown 文本，可异步渲染为图片
  • C++ 单元格：C++ 代码，按 main() 函数边界自动分组编译，同组 cell 合并编译运行，互不干扰
  • Python 单元格：Python 代码，持久化进程执行，跨 cell 共享命名空间

操作方式

SMD 编辑支持双模式系统：

  • 命令模式（紫色边框 #C586C0）：键盘导航，Esc 从编辑模式切换至命令模式
  • 编辑模式（蓝色边框 #0078D4）：文本编辑

快捷键：

  • A / B（命令模式）：在当前单元格上方/下方插入新单元格（新建时自动弹出语言选择菜单）
  • Delete（命令模式）：删除当前单元格（至少保留一个）
  • Ctrl+K：弹出语言选择菜单（Markdown/C++/Python）
  • Ctrl+Shift+-：在光标处分割单元格
  • Ctrl+Enter：执行当前单元格（不跳转）
  • Shift+Enter：执行当前单元格并跳转至下一个
  • Ctrl+T：.md ↔ .smd 双向转换
  • Ctrl+D（编辑模式）：切换诊断面板

输出与诊断

  • 每个单元格下方有独立的输出显示区域（stdout/stderr），自适应高度（1~15 行，上限 1000 行），超出时保留前 1000 行并提示隐藏行数
  • C++ 单元格按 main() 边界自动分组，同组 cell 合并编译运行，不同程序组互不干扰
  • Python 单元格使用持久化进程（python_executor.py），跨 cell 共享命名空间，每 cell 独立捕获 stdout/stderr
  • 输出自动持久化至 .smd 文件元数据中（base64 编码 + JSON），重新打开时自动恢复
  • 执行期间禁止增删单元格，Ctrl+C 可终止执行

LSP 代码智能

SMD 文件中的 C++/Python 单元格共享 LSP 后端（每种语言一个 clangd/Jedi 进程，而非每 cell 一个），通过 虚拟文档拼接 技术实现跨 cell 类型解析：

  • 同语言 cell 拼接为有效源文件，行号自动映射，跨 cell 类型解析
  • C++ 按 main() 分组，仅向 clangd 发送当前活动组代码，避免多 main() 冲突
  • 代码补全（Ctrl+I / 自动触发）、悬停类型提示、函数签名帮助
  • 诊断波浪线（红色错误 / 黄色警告），cell 头部标签显示错误计数
  • 切换 cell 时自动缓存/恢复诊断结果

代码编译运行

在代码编辑模式下，支持编译运行 C/C++ 和 Python 文件。自动检测系统中可用的编译器（优先 g++，其次 MSVC cl.exe）。非代码文件时工具栏运行按钮自动隐藏，快捷键同步失效。切换至非代码文件时自动关闭输出面板，若进程运行中则先终止后隐藏。

快捷键

快捷键功能
F5编译并运行（C/C++ 编译后运行，Python 直接运行）
F6仅编译（C/C++；Python 提示不需要编译）
F7仅运行（无可执行文件时自动转为编译运行）
Ctrl+Break终止正在运行的编译或程序

输出面板

编译和运行的结果显示在编辑器下方的底部面板（BottomPanel）中，含「输出」和「诊断」两个标签页。输出面板嵌入右侧分割区，不延伸至文件树区域。支持：

  • stdout 白色显示、stderr 红色显示
  • 运行时支持标准输入交互（直接输入、粘贴多行，粘贴时逐行交错显示输出）
  • 底部状态标签（编译成功绿色/失败红色）+ 终止按钮 + 清除按钮
  • 隐藏输出面板时若进程正在运行则自动终止
  • 首次显示时自动调整高度为窗口高度的 1/3

OpenJudge IDE 模式

在 OpenJudge 题目 IDE 模式下，编译运行使用 IDE 内置编辑器中的代码，语言由 IDE 语言选择器（C++/Python）决定，其余流程与普通代码编辑模式一致。

搜索与大纲

全文搜索

通过左侧活动栏的搜索按钮（Ctrl+Shift+F）打开全文搜索面板。搜索面板为持久侧边栏（点击外部不自动隐藏），显示时自动聚焦搜索输入框。

  • 异步后台搜索：在后台线程执行目录遍历 + 文件读取 + 行匹配，不阻塞 UI。支持取消令牌 + 代数计数器并发管理，新搜索自动取消旧搜索
  • 搜索输入 300ms 防抖，避免频繁磁盘扫描
  • 结果显示文件名、行号和上下文片段（自动截断并对齐匹配关键词位置）
  • 单击结果：以临时预览标签页打开文件并高亮匹配关键词
  • 双击结果：永久打开文件并高亮匹配关键词，编辑预览内容自动转为永久标签
  • 结果上限可配置（每文件最大匹配数、总结果上限、片段最大长度），设置修改实时生效
  • 与评测面板共用左侧 dock 区域（显示一个时自动隐藏另一个）

大纲面板

通过右侧面板的「大纲」标签页（Ctrl+Shift+O）打开。自动解析当前 Markdown 文档中的全部标题（# ~ ######，跳过围栏代码块）。

  • 按层级缩进显示，缩进量相对于文件中最小标题层级（非绝对从 h1 起算）
  • 文字颜色主题自适应（侧边栏前景色与背景色混合，h1 最深 h6 最浅），h1/h2 加粗
  • 长标题超出右边界时自动显示省略号，hover 显示完整标题
  • 点击标题可精准跳转至对应位置并高亮目标行（1 秒自动消失），主题切换时高亮颜色同步更新
  • 切换标签页、保存文件时自动刷新
  • 非 .md 文件时面板自动清空，关闭面板时自动清除高亮

双向链接与标签

WikiLink 双向链接

使用 [[文件名]] 语法创建文件之间的双向链接。预览模式下自动识别为超链接，点击可跳转至对应文件。

  • 自动补全：输入 [[ 时自动弹出文件名列表，Tab 补全并自动闭合 ]]
  • 自动创建：若链接的文件不存在，支持一键自动创建
  • 重命名同步：文件重命名时，自动更新所有引用文件中的双向链接文本（通过文件索引多级搜索策略匹配）
  • 文件索引：后台线程异步构建文件索引，支持代际计数器保护防止过期结果覆盖

标签系统

使用 #tag 语法为文档添加标签（仅 .md / .markdown 文件）。

  • 自动补全：输入 # 时自动弹出已有标签列表，Tab 补全
  • 可点击标签：预览模式下 #tag 自动转为可点击链接
  • 标签导航：通过右侧面板的「标签」标签页（Ctrl+Shift+T）浏览所有标签及关联文件
  • 支持 Unicode（中文/日文/韩文），自动跳过代码块避免误匹配

反向链接面板

通过右侧面板的「反链」标签页（Ctrl+Shift+B）查看引用当前文件的来源文件列表。包含：

  • 点击来源文件可快速跳转
  • 自动扫描 [[wikilink]] 引用关系，同一文件内多次相同链接自动去重
  • 文件保存后增量更新反链索引
  • 文件重命名/删除时自动维护索引一致性

历史记录

通过右侧面板的「历史」标签页（Ctrl+H）访问最近打开的文件列表。

  • 列表上限 50 条，自动去重（已存在则置顶）
  • 顶部为最新打开的文件
  • 点击可快速重新打开，若文件在当前目录外则自动切换根目录
  • 文件删除后自动清理对应历史条目；点击已不存在文件的条目时自动移除
  • 底部「清空历史记录」按钮
  • 右侧面板点击外部可自动隐藏（通过焦点变化 + 防误触定时器实现）

评测系统

提供本地评测（OJ 风格）和 OpenJudge 在线评测集成。

本地评测

通过活动栏的评测按钮（Ctrl+Shift+J）打开评测面板。在代码编辑模式下，可选择测试用例文件夹，一键批量运行所有测试用例。

  • 结果状态：AC（通过，绿色）、WA（答案错误，红色）、RE（运行时错误，橙色）、TLE（超时，蓝色）、MLE（超内存，紫色），结果列按状态码着色
  • 耗时/内存：每个用例的耗时(ms)和峰值内存(KB)检测
  • 详情查看：点击失败行查看预期输出与实际输出对比
  • 预热机制：编译后先运行一次消除冷启动计时偏差
  • 时间限制：每个用例 1000ms
  • 内存限制：64MB（65536KB），三重捕获策略（启动时同步 + 100ms 轮询 + 退出时补充）确保准确检测
  • 支持 C/C++ 和 Python 评测

错题本

本地评测或 OpenJudge 提交失败时自动记录到错题本，集成在 AI 面板中。

  • 记录输入数据、预期输出、实际输出、状态码、耗时内存等详细信息
  • 支持 AI 自动分析错误原因
  • 支持按状态筛选和关键词搜索
  • AI 面板中通过徽章计数显示未处理错误数

OpenJudge 集成

通过评测面板的「从OpenJudge获取」按钮在主窗口标签页中打开 OpenJudge 题目浏览窗口（非独立窗口），可登录或跳过登录直接浏览。

功能概述

  • 作业浏览：三级导航（作业列表 → 题目列表 → 题目详情），已结束作业支持分页浏览
  • 题目详情：左侧章节导航，右侧渲染题目内容（深色主题），全面接入主题系统
  • 样例提取：点击「选择此题目」自动提取样例输入/输出并写入临时缓存目录
  • 自动登录：支持保存凭据并自动登录，退出登录时清除 Cookie 并清除凭据

IDE 模式

题目详情页顶部工具栏新增「IDE」切换按钮，进入 IDE 模式后页面中间为水平分隔条：

  • 左侧为题目内容，右侧为嵌入式代码编辑器（完整支持语法高亮、LSP 补全、悬停提示、签名帮助、诊断波浪线）
  • 分隔条拖拽范围限制 3:7 ~ 7:3
  • 语言选择下拉框（C++/Python），自动根据题目可用语言和上次切换记录智能选择
  • 代码自动缓存至临时目录，退出 IDE 模式或切换语言时自动保存，重新进入时自动恢复
  • IDE 模式下编译运行（F5/F6/F7）、本地评测、OpenJudge 提交均使用编辑器当前内容

代码提交

评测面板的「提交到OpenJudge」按钮可将当前代码文件直接提交到选定的题目。

  • 自动映射文件扩展名到对应语言（.c → GCC，.cpp → G++，.py → Python3）
  • 提交前检查登录状态、代码有效性、题目选择状态及作业是否进行中
  • IDE 模式下提交内容来自内置编辑器，语言由语言选择器指定

提交结果面板

提交后自动显示评测结果面板（占右侧分割区 1/3 高度），大号彩色状态文字：

  • AC 绿色、WA 红色、TLE 蓝色、MLE 紫色、RE 红色、PE 深橙、OLE 粉红、CE 橙色
  • 显示用时(ms)和内存(KB)，CE 时展示编译错误日志
  • 接入主题系统，切换主题时颜色实时同步
  • 非 AC 非 CE 的提交失败自动触发本地复测并记入错题本

AI 助手

通过活动栏的 AI 按钮（Ctrl+Shift+A）打开 AI 助手面板。支持 Anthropic Claude 和 OpenAI 兼容 API。

对话功能

  • 在输入框中输入问题或指令，按发送按钮或 Enter 提交
  • 支持上下文感知操作按钮（改进、总结、解释等），根据编辑器当前模式动态显示
  • AI 上下文管理器自动收集编辑器上下文（模式、文件、选区、语言）
  • 流式响应显示（SSE），80ms 防抖合并渲染，增量 HTML 更新
  • 多轮对话历史管理，支持 Token 感知的窗口裁剪

历史对话

  • 自动持久化对话历史，支持按当前文件过滤对话列表
  • 历史列表以绿色圆点标记活跃对话
  • 可加载历史对话继续交流

错题本

AI 面板集成错题本标签页，显示评测失败记录：

  • 条目卡片：状态徽章（按状态码着色）、题目名、源文件名、时间戳
  • 展开详情：状态码及其中文标签、耗时内存、输入输出对比区
  • 一键 AI 分析按钮，自动生成错误原因分析
  • 支持按状态筛选、关键词搜索、标记已阅

配置

在设置面板的「AI 服务」页面配置：

  • AI 供应商（Anthropic / OpenAI）
  • API 端点
  • API 密钥
  • 模型名称
  • 最大 Token 数
  • 系统提示词

设置与快捷键

设置面板

通过活动栏的设置按钮（Ctrl+,）打开悬浮式设置面板，背景自动变暗。支持拖拽标题栏移动和边缘拖拽调整大小，右上角关闭按钮或再次按快捷键关闭。

设置分类：

分类可设置项
编辑器默认缩放比例（50%~300%，滑块 + 数字输入）、缩进宽度、编辑器字体、字号、自动保存开关、自动保存间隔
外观编辑器背景色、前景色、行号背景/前景、当前行高亮、搜索高亮、标签页高度（20–40px）、等宽标签页开关
文件树文件树行高
输出面板字号、最大行数
预览分屏防抖延迟（100~2000ms）、分屏比例（30%~70%）
搜索每文件最大匹配数、总结果上限、片段最大长度
快捷键自定义所有快捷键（点击录制，Delete/Backspace 清除，Esc 取消）
AI 服务供应商、端点、API 密钥、模型、最大 Token 数、系统提示词

快捷键自定义

在设置面板的「快捷键」页面支持交互式录制：

  • 点击快捷键区域进入录制状态
  • 按下目标组合键完成录制
  • Delete / Backspace 清除当前快捷键
  • Esc 取消录制
  • 快捷键允许冲突（类似 Minecraft 风格），冲突的快捷键以红色文字标出，由用户自行决定是否修改
  • 所有修改实时生效，自动持久化至 config.ini
  • 「恢复默认设置」一键重置所有快捷键和配置

自动保存

编辑器自动保存机制，默认开启（30 秒间隔）。文件加载后及手动保存后自动启动定时器，有修改时自动写入文件。可在设置面板中通过开关控件实时开启/关闭。

无边框窗口

隐藏系统原生标题栏，工具栏上移至标题栏位置：

  • 左侧「文件」菜单按钮（文字 + SVG 箭头）
  • 中间拖拽区域（双击最大化/还原窗口）
  • 右侧运行按钮组（仅代码文件可见）+ 最小化/最大化/关闭按钮（Windows 11 原生图标）
  • 支持拖拽空白区域移动、边缘 10px 缩放、Aero Snap 贴靠
