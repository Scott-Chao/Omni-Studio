Smart Markdown — 帮助文档

Smart Markdown 是一款支持 Markdown 编辑、代码编辑、双向链接、SMD 单元格编辑、代码编译运行、本地评测、AI 助手等功能的全功能文本编辑器。

快速入门

1. 打开文件夹：点击工具栏「文件 → 打开目录」或通过快捷键，选择一个文件夹作为工作区根目录。文件树将显示该目录下的所有文本文件。

2. 打开文件：在左侧文件树中点击任意文件即可打开。支持同时打开多个文件，以标签页形式呈现。

3. 编辑与保存：直接编辑文件内容，支持 Ctrl+S 保存。新建文件使用 Ctrl+N。

4. 预览 Markdown：打开 .md 文件后，点击工具栏「预览」按钮或 Ctrl+Shift+P 进入渲染预览模式。

5. 代码编辑与运行：打开 .cpp、.py 等代码文件，自动切换为代码编辑模式（语法高亮、行号、代码补全）。按 F5 编译运行。

6. 探索更多：左侧活动栏提供搜索、AI 助手、设置、评测等功能入口。右侧面板提供历史记录、大纲、标签、反链等功能。

文件管理

文件树

左侧文件树以树状结构展示当前工作区根目录下的所有文件和文件夹。支持的操作包括：

  • 点击：打开文件（自动切换标签页）
  • 右键：弹出上下文菜单，支持新建文件、新建文件夹、重命名、删除
  • 拖拽：在文件树内拖拽移动文件或文件夹
  • Delete 键：选中文件/文件夹后直接删除

文件操作

  • 新建文件（Ctrl+N）：在文件树中或通过快捷键创建新文件
  • 保存（Ctrl+S）：保存当前编辑的文件
  • 另存为（Ctrl+Shift+S）：将当前文件另存为新文件
  • 关闭：关闭标签页时若文件有未保存的修改，会提示保存

面包屑路径

文件树顶部显示当前根目录的完整路径，每个路径段可点击快速跳转。路径过长时自动换行。

文件类型支持

支持 40+ 种常见文本文件格式，包括：.md、.txt、.c、.cpp、.py、.js、.html、.css、.json、.xml 等。所有文件以纯文本形式呈现在编辑器中。

标签页联动

切换标签页时，文件树自动选中对应的文件，并展开折叠的父级目录，确保文件在树中可见。

编辑器

编辑器根据文件扩展名自动切换为合适的编辑模式，共支持六种模式：

Markdown 编辑模式（.md, .txt 等）

使用 WikiLinkTextEdit（QTextEdit 子类），支持：

  • [[wikilink]] 自动补全：输入 [[ 时弹出文件名列表，方向键选择，Tab 补全并自动闭合 ]]
  • #tag 自动补全：输入 # 时弹出已有标签列表，Tab 补全
  • Markdown 语法高亮

代码编辑模式（.cpp, .py, .js 等）

使用 CodeEditor（QPlainTextEdit 子类），提供 IDE 风格功能：

  • 语法高亮：深色主题，C/C++（关键字蓝色、类型青色、字符串橙色、注释绿色），Python（类似配色）
  • 行号显示：左侧灰色行号区域
  • 自动缩进：Enter 自动保持缩进，智能处理 { } 和 Python 冒号缩进
  • 括号补全：输入 { ( [ " ' 时自动插入匹配对
  • 智能退格：退格键以缩进步长为单位删除前导空白
  • 行注释切换：Ctrl+/ 切换注释/取消注释
  • 缩进调整：Ctrl+[ 向左缩进，Ctrl+] 向右缩进
  • 代码补全：Ctrl+I 或输入 . :: -> 自动触发。C/C++ 使用 clangd LSP 后端，Python 使用 Jedi 后端
  • 悬停类型提示：鼠标悬停在变量/函数上 400ms 后显示类型信息
  • 函数签名帮助：输入 ( 后弹出函数签名提示
  • 诊断波浪线：红色错误 / 黄色警告波浪下划线，鼠标悬停显示诊断信息
  • 诊断面板：Ctrl+D 切换底部诊断面板（与输出面板合并）

其他模式

  • PDF 阅读模式：打开 .pdf 文件时自动切换，使用 QPdfView + QPdfDocument 渲染
  • SMD 编辑模式：打开 .smd 文件时自动切换，单元格分块编辑（详见 SMD 编辑器章节）

字体缩放

支持对编辑器和 Markdown 预览进行字体缩放，范围 50%~300%：

  • 工具栏缩放按钮（− / 百分比 / + / 重置）
  • Ctrl+= 放大、Ctrl+- 缩小、Ctrl+0 重设为默认缩放比例
  • Ctrl+鼠标滚轮 缩放
  • 触控板缩放手势

Markdown 预览

打开 .md 文件时可切换至渲染预览模式，基于 WebEngine 渲染引擎。

全屏预览

点击工具栏「预览」按钮或按 Ctrl+Shift+P 进入全屏预览模式。支持以下渲染特性：

  • GitHub Flavored Markdown：标准 Markdown 语法（标题、列表、表格、代码块、引用等）
  • LaTeX 数学公式：$...$ 行内公式、$$...$$ 块级公式（基于 KaTeX）
  • Mermaid 图表：```mermaid 代码块渲染为流程图、时序图、甘特图等
  • 代码块语法高亮：使用 C++ 端预处理方案，复用编辑器语法高亮规则
  • WikiLink：[[文件名]] 自动识别为超链接，点击可跳转至对应文件
  • Tag：#tag 自动识别为可点击标签
  • 代码块运行按钮：预览代码块右上角显示 ▶ 运行按钮，点击可编译运行

分屏预览

点击工具栏「分屏」按钮或按 Ctrl+P 进入分屏预览模式。编辑器区域被竖直分隔条分为左右两部分：左侧为 Markdown 源码，右侧为渲染预览。

  • 全屏预览与分屏预览互斥（开启一个自动关闭另一个）
  • 右侧预览采用 500ms 防抖延迟更新（可在设置面板中调节）
  • 分屏比例可在设置面板中调节（30%~70%）
  • 切换文件时自动记忆各标签页的预览状态

提示：非 .md 文件下预览按钮自动隐藏且快捷键无效。

SMD 编辑器

.smd 文件采用 ---smd:<type> 分隔符的单元格分块格式，类似 Jupyter Notebook 的交互模式。

单元格类型

  • Markdown 单元格：Markdown 文本，可渲染为图片
  • C++ 单元格：C++ 代码，按 main() 函数边界自动分组编译
  • Python 单元格：Python 代码，持久化进程执行，跨 cell 共享命名空间

操作方式

SMD 编辑支持双模式系统：

  • 命令模式（紫色边框 #C586C0）：键盘导航，Esc 从编辑模式切换至命令模式
  • 编辑模式（蓝色边框 #0078D4）：文本编辑

快捷键：

  • A / B（命令模式）：在当前单元格上方/下方插入新单元格
  • Delete（命令模式）：删除当前单元格（至少保留一个）
  • Ctrl+K：弹出语言选择菜单（Markdown/C++/Python）
  • Ctrl+Shift+-：在光标处分割单元格
  • Ctrl+Enter：执行当前单元格（不跳转）
  • Shift+Enter：执行当前单元格并跳转至下一个
  • Ctrl+T：.md ↔ .smd 双向转换
  • Ctrl+D（编辑模式）：切换诊断面板

输出与诊断

  • 每个单元格下方有独立的输出显示区域（stdout/stderr），自适应高度（1~15 行，上限 1000 行）
  • C++ 单元格按 main() 边界自动分组，同组 cell 合并编译，互不干扰
  • Python 单元格使用持久化进程（python_executor.py），跨 cell 共享命名空间
  • 输出自动持久化至 .smd 文件元数据中，重新打开时自动恢复

LSP 代码智能

SMD 文件中的 C++/Python 单元格共享 LSP 后端（每种语言一个 clangd/Jedi 进程），提供：

  • 代码补全（Ctrl+I / 自动触发）
  • 悬停类型提示
  • 函数签名帮助
  • 诊断波浪线（红色错误 / 黄色警告）
  • 虚拟文档拼接技术实现跨 cell 类型解析

代码编译与运行

在代码编辑模式下，支持编译运行 C/C++ 和 Python 文件。非代码文件时工具栏运行按钮自动隐藏，快捷键同步失效。

快捷键

快捷键功能
F5编译并运行（C/C++ 编译后运行，Python 直接运行）
F6仅编译（C/C++；Python 提示不需要编译）
F7仅运行（无可执行文件时自动转为编译运行）
Ctrl+Break终止正在运行的编译或程序

输出面板

编译和运行的结果显示在编辑器下方的输出面板中（嵌入右侧分割区，不延伸至文件树区域）。支持：

  • stdout 白色显示、stderr 红色显示
  • 运行时支持标准输入交互（直接输入、粘贴多行）
  • 底部状态标签（编译成功绿色/失败红色）+ 终止按钮 + 清除按钮
  • 隐藏输出面板时若进程正在运行则自动终止

搜索与大纲

全文搜索

通过左侧活动栏的搜索按钮（Ctrl+Shift+F）打开全文搜索面板。在当前根目录的所有文本文件中检索关键词。

  • 搜索输入 300ms 防抖，避免频繁磁盘扫描
  • 结果显示文件名、行号和上下文片段
  • 点击结果跳转至对应文件并高亮匹配关键词
  • 结果上限可配置（每文件最大匹配数、总结果上限、片段最大长度）
  • 与评测面板共用左侧 dock 区域（显示一个时自动隐藏另一个）

大纲面板

通过右侧面板的「大纲」标签页（Ctrl+Shift+O）打开。自动解析当前 Markdown 文档中的全部标题（# ~ ######，跳过围栏代码块）。

  • 按层级缩进显示，h1 最亮 h6 逐级变暗
  • h1/h2 加粗
  • 点击标题精准跳转至对应位置
  • 切换标签页、保存文件时自动刷新
  • 非 .md 文件时面板自动清空

双向链接与标签

WikiLink 双向链接

使用 [[文件名]] 语法创建文件之间的双向链接。预览模式下自动识别为超链接，点击可跳转至对应文件。

  • 自动补全：输入 [[ 时自动弹出文件名列表，Tab 补全并自动闭合 ]]
  • 自动创建：若链接的文件不存在，支持一键自动创建
  • 重命名同步：文件重命名时，自动更新所有引用文件中的双向链接文本
  • 路径匹配：支持多级搜索策略，通过文件名索引智能解析路径

标签系统

使用 #tag 语法为文档添加标签。

  • 自动补全：输入 # 时自动弹出已有标签列表，Tab 补全
  • 可点击标签：预览模式下 #tag 自动转为可点击链接
  • 标签导航：通过右侧面板的「标签」标签页（Ctrl+Shift+T）浏览所有标签及关联文件

反向链接面板

通过右侧面板的「反链」标签页（Ctrl+Shift+B）查看引用当前文件的来源文件列表。包含：

  • 点击来源文件可快速跳转
  • 自动扫描 [[wikilink]] 引用关系
  • 文件保存后增量更新反链索引
  • 文件重命名/删除时自动维护索引一致性

历史记录

通过右侧面板的「历史」标签页（Ctrl+H）访问最近打开的文件列表。

  • 列表上限 50 条，自动去重
  • 顶部为最新打开的文件
  • 点击可快速重新打开
  • 文件删除后自动清理对应历史条目
  • 底部「清空历史记录」按钮

评测系统

提供本地评测（OJ 风格）和 OpenJudge 在线评测集成。

本地评测

通过活动栏的评测按钮（Ctrl+Shift+J）打开评测面板。在代码编辑模式下，可选择测试用例文件夹，一键批量运行所有测试用例。

  • 结果状态：AC（通过）、WA（答案错误）、RE（运行时错误）、TLE（超时）、MLE（超内存），结果列按状态码着色
  • 耗时/内存：每个用例的耗时(ms)和峰值内存(KB)检测
  • 详情查看：点击失败行查看预期输出与实际输出对比
  • 预热机制：编译后先运行一次消除冷启动计时偏差
  • 时间限制：每个用例 1000ms
  • 内存限制：64MB（65536KB），三重捕获策略确保准确检测
  • 支持 C/C++ 和 Python 评测

OpenJudge 集成

通过评测面板的「从OpenJudge获取」按钮打开独立浏览窗口，可登录 OpenJudge 或跳过登录直接浏览题目。

功能概述

  • 作业浏览：三级导航（作业列表 → 题目列表 → 题目详情），已结束作业支持分页浏览
  • 题目详情：左侧章节导航，右侧渲染题目内容（深色主题）
  • 样例提取：点击「选择此题目」自动提取样例输入/输出并写入临时缓存目录
  • 自动登录：支持保存凭据并自动登录

代码提交

评测面板的「提交到OpenJudge」按钮可将当前代码文件直接提交到选定的题目。

  • 自动映射文件扩展名到对应语言（.c → GCC，.cpp → G++，.py → Python3）
  • 提交前检查登录状态、代码有效性、题目选择状态及作业是否进行中
  • 提交后自动显示评测结果面板（暗色主题），大号彩色状态文字
  • AC 绿色、WA 红色、TLE 蓝色、MLE 紫色、RE 红色、PE 深橙、OLE 粉红、CE 橙色
  • 显示用时(ms)和内存(KB)，CE 时展示编译错误日志

AI 助手

通过活动栏的 AI 按钮（Ctrl+Shift+A）打开 AI 助手面板。支持 Anthropic Claude 和 OpenAI 兼容 API。

使用方式

  • 在输入框中输入问题或指令，按发送按钮或 Enter 提交
  • 支持上下文感知操作按钮（改进、总结、解释等），根据编辑器当前模式动态显示
  • AI 上下文管理器自动收集编辑器上下文（模式、文件、选区、语言）
  • 流式响应显示（SSE）

配置

在设置面板的「AI 服务」页面配置：

  • AI 供应商（Anthropic / OpenAI）
  • API 端点
  • API 密钥
  • 模型名称
  • 最大 Token 数
  • 系统提示词

设置与快捷键

设置面板

通过活动栏的设置按钮（Ctrl+,）打开悬浮式设置面板。支持拖拽标题栏移动和边缘拖拽调整大小。

设置分类：

分类可设置项
编辑器默认缩放比例（50%~300%）、缩进宽度、编辑器字体、字号
外观编辑器背景色、前景色、行号背景/前景、当前行高亮、搜索高亮
输出面板字号、最大行数
预览分屏防抖延迟（100~2000ms）、分屏比例（30%~70%）
搜索每文件最大匹配数、总结果上限、片段最大长度
快捷键自定义所有快捷键（点击录制，Delete/Backspace 清除，Esc 取消）
AI 服务供应商、端点、API 密钥、模型、最大 Token 数、系统提示词

快捷键自定义

在设置面板的「快捷键」页面支持交互式录制：

  • 点击快捷键区域进入录制状态
  • 按下目标组合键完成录制
  • Delete / Backspace 清除当前快捷键
  • Esc 取消录制
  • 冲突检测：弹窗提示覆盖或取消，覆盖时自动清空冲突操作的快捷键
  • 所有修改实时生效，自动持久化至 config.ini
  • 「恢复默认设置」一键重置所有快捷键

自动保存

编辑器自动保存机制，默认开启（30 秒间隔）。文件加载后及手动保存后自动启动定时器。可在设置面板中开启/关闭。
