## Build and Run

### Prerequisites
- **Environment**: MUST run from "x64 Native Tools Command Prompt for VS 2022".
- **System**: Qt 6.11.0 + MSVC 2022.
- **Build Dir**: `./build/Desktop_Qt_6_11_0_MSVC2022_64_bit-Debug`
- **Output**: `./release/smart-markdown.exe`

### Commands
- **Configure**: `qmake.exe -r smart-markdown.pro` (re-run when QRC changes)
- **Build**: `jom.exe -f Makefile.Release -j22` (or `nmake` as fallback)
- **Clean**: `jom.exe -f Makefile.Release clean` (or `nmake clean`)
- **Deploy**: `windeployqt release/smart-markdown.exe --qmldir .`

## Code Architecture

The app is a single-window Markdown editor (Qt Widgets). All source files are at the project root. The architecture follows a flat component layout with `MainWindow` as the central coordinator.

### Key Data Flow

- **File opening**: `FileExplorerWidget::fileClicked` вҶ’ `MainWindow::onFileSelected` вҶ’ `TabManager::openFile` вҶ’ returns `EditorWidget*`. Mode auto-detected by `EditorWidget::loadFile()`: if extension maps to a known language in `LanguageUtils::languageForExtension()`, switches to `CodeEdit` (page 2 with `CodeEditor` + syntax highlighter); otherwise `MarkdownEdit` (page 0 with `WikiLinkTextEdit`).
- **Code editing**: `CodeEditor::keyPressEvent` routes EnterвҶ’`handleAutoIndent` (auto-indent, `{|}` splits to 3 lines), TabвҶ’`handleTabKey` (insert 4 spaces / indent selection), BackspaceвҶ’`handleBackspaceIndent` (delete to tab-stop in leading whitespace) then `handleBackspacePairRemoval` (delete empty `{}()` etc.), bracket keysвҶ’`handleBracketCompletion` (auto-pair, skip in string/comment) or `handleClosingBracketSkip` (skip over existing bracket). Search highlights stored in `m_searchHighlights` and merged with current-line highlight in `highlightCurrentLine()`.
- **Bidirectional links**: `[[ж–Үд»¶еҗҚ]]` syntax вҶ’ regex in `EditorWidget::refreshPreview` converts to `<a href="wikilink:...">` tags вҶ’ `PreviewPage::acceptNavigationRequest()` intercepts clicks on `wikilink:` scheme вҶ’ emits `wikiLinkClicked` вҶ’ `MainWindow::onWikiLinkClicked` вҶ’ `MainWindow::findWikiTarget` (multi-level filename search) вҶ’ opens or prompts create
- **Preview code block run**: preview-template.html marked renderer adds в–¶ Run button on `python`/`cpp`/`c`/`py`/`cc`/`cxx` code blocks вҶ’ click stores code in `window._runCodeData` вҶ’ navigates to `runblock:execute` вҶ’ `PreviewPage::acceptNavigationRequest()` intercepts `runblock:` scheme вҶ’ `runJavaScript` reads stored data вҶ’ emits `runCodeBlockRequested` вҶ’ `MainWindow::onCodeBlockRequested` вҶ’ `LanguageUtils::normalizeCodeFenceLanguage()` maps fence lang вҶ’ `saveCodeBlockToTempFile()` writes to temp вҶ’ `ProcessRunner::startRunPython()` or `startCompileAndRun()` вҶ’ `OutputPanel` shows output. Both full-preview and split-preview WebEngine views share identical PreviewPage subclasses and signal forwarding.
- **File indexing**: `MainWindow::m_fileIndex` (QMap<filename_no_ext, QStringList<full_paths>>) вҖ” rebuilt asynchronously on folder change via `startAsyncIndexBuild()` using `QThread::create()` with cancellation token (`std::shared_ptr<std::atomic<bool>>`) and generation counter (`std::atomic<uint64_t> m_scanId`) to reject stale results. Synchronous rebuild retained for single-file incremental updates on rename/delete/move. Scans use `TextFileUtils::scanNameFilters()` for 40+ text extensions.
- **Backlinks**: `TabManager::currentChanged` вҶ’ `MainWindow::refreshBacklinks` вҶ’ `BacklinkIndex::backlinksFor(currentFilePath)` вҶ’ `BacklinksPanel::showBacklinks`. Full index built asynchronously via `BacklinkIndex::buildFromPath()` (static, thread-safe) + `setData()` (main-thread swap). Per-file incremental updates via `rebuildFile` / `onFileRenamed` / `onFileDeleted` remain synchronous.
- **Tags**: `#tag` syntax in `.md` files extracted by `TagIndex` (static `buildFromPath()`, async Phase 3). `TagPanel` shows tag list вҶ’ click tag вҶ’ file list вҶ’ open file. Preview `#tag` вҶ’ `<a href="tag:...">` via `processTagsForPreview()`. Autocomplete in edit mode via `WikiLinkTextEdit#` detection. Async build in startup scan; incremental updates on save/rename/delete. `Ctrl+Shift+T` toggles panel, auto-hides on outside click.
- **Rename + wiki link update**: `FileExplorerWidget::fileRenamed` вҶ’ `MainWindow::onFileMovedOrRenamed` вҶ’ `onFileRenamedInIndex` captures source files linking to old path, updates index, then `updateWikiLinksAfterRename` rewrites `[[oldName]]` вҶ’ `[[newName]]` in each source file (using `replaceWikiLinkText` with exact wiki-link regex matching). Open editors with unsaved changes are handled by reading from `editor->toPlainText()` instead of disk.
- **Save**: `MainWindow::saveFile` вҶ’ `EditorWidget::saveFile` (existing) or `EditorWidget::saveAsFile` (new = uses remembered save-as dir)
- **Split-pane preview**: `MainWindow` toolbar `Ctrl+P` вҶ’ `EditorWidget::setSplitPreviewMode(true)` вҶ’ lazy-creates `QSplitter` (page 3) with `m_textEdit` (left) + second `QWebEngineView` (right). On text change вҶ’ `onTextChanged()` starts 500ms debounce timer вҶ’ `onSplitDebounceTimeout()` checks content diff вҶ’ `updateSplitPreviewContentNow()`. Mutual exclusion with full preview handled bidirectionally. Open/close handlers in MainWindow sync toolbar button states via `updateSplitPreviewActionState()`.
- **Zoom**: any zoom action вҶ’ `EditorWidget::setZoomFactor` вҶ’ `applyZoom` (adjusts QTextEdit fonts and all QWebEngineView zoom factors including split preview) вҶ’ emits `zoomFactorChanged` вҶ’ `MainWindow::updateZoomLabel`
- **Outline panel**: `MainWindow::refreshOutline` on tab switch/save вҶ’ `extractHeadingsFromContent` (regex `^(#{1,6})\\s+(.+)$`, skips code blocks) вҶ’ `OutlinePanel::showHeadings`. Click heading вҶ’ `EditorWidget::scrollToLine`. Only for `.md` files. Toggle `Ctrl+Shift+O`, auto-hide on outside click.
- **Full-text search**: `SearchPanel::performSearch` (300ms debounce on text input) вҶ’ `QDirIterator` + `TextFileUtils::scanNameFilters` scans all text files вҶ’ `QTextStream::readLine` streaming match per line вҶ’ `SearchPanel::resultClicked` вҶ’ `MainWindow::onSearchResultClicked` вҶ’ `TabManager::openFile` вҶ’ `EditorWidget::scrollToLine` (finds line via `QTextBlock`, highlights all matches with `setExtraSelections` gold background)
- **WikiLink autocomplete**: `WikiLinkTextEdit::keyPressEvent` detects `[[` вҶ’ `updateCompleter` checks cursor is inside open `[[...` on current line вҶ’ `QCompleter` with `QStringListModel` (populated from `m_fileIndex.keys()`) shows matching filenames prefix-filtered вҶ’ `cursorPositionChanged` + arrow keys trigger re-evaluation вҶ’ Tab inserts `[[filename]]`, cursor after `]]`. File list propagated from `MainWindow::updateCurrentEditorCompletions` called at end of `buildFileIndex()` and on tab switch.
- **Compile & run**: toolbar F5/F6/F7 вҶ’ `MainWindow::onCompileAndRun`/`onCompile`/`onRun` вҶ’ `ProcessRunner` spawns `QProcess` (g++ or MSVC for C/C++; python for `.py`). stdout/stderr streamed to `OutputPanel` in real-time (no `.trimmed()`, raw format preserved). For C/C++: on compile success, `.exe` is launched as a second `QProcess`. For Python: `startRunPython` invokes the interpreter directly. Exit code shown in panel status bar. `CompilerUtils` detects available compilers and python interpreter. Buttons auto-disable while running; Stop (Ctrl+Break) kills the process. Unsaved files are auto-saved to temp before compiling/running. **stdin support**: `OutputPanel` captures keyboard input via `eventFilter` when process is running (characters echoed + buffered, EnterвҶ’`sendInput`вҶ’`ProcessRunner::writeInput`вҶ’`QProcess::write(text+"\n")`). Paste (Ctrl+V/right-click) splits multi-line text and delivers line-by-line via 20ms timer with `processEvents()` between lines for interleaved output; last line without trailing `\n` is echoed and buffered (editable, send on Enter). Ctrl+C emits `stopRequested()`. Compilation phase blocks input entirely (no cursor, no selection), 50ms input enable delay on run start, focus returns to editor on process stop.
- **Local Judge**: `JudgePanel::runAllRequested` вҶ’ `MainWindow::onJudgeRunAll` (saves code, shows dock, validates) вҶ’ `JudgePanel::runJudge(filePath)` вҶ’ `JudgeEngine::start()` discovers `.in`/`.out` pairs (skips empty `.out`) вҶ’ compile вҶ’ warmup run (empty stdin) вҶ’ runs each test case sequentially with 1000ms timeout + tripleвҖ‘capture memory monitoring (`captureMemory()` at start/exit + 100ms poll). Output trimmed line-by-line comparison. Results in table: AC (green), WA (red), TLE (blue), MLE (purple), RE (orange). Click failed row for expected vs actual diff.
- **OpenJudge integration**: `JudgePanel "д»ҺOpenJudgeиҺ·еҸ–" button` вҶ’ `emit openJudgeRequested()` вҶ’ `MainWindow::onOpenJudgeRequested()` вҶ’ creates/raises singleton `OpenJudgeWindow` (QPointer managed) вҶ’ `Crawler` fetches OpenJudge pages via JSON API (`/api/auth/login/`, PHP-based) вҶ’ navigate homework list вҶ’ problem list вҶ’ problem detail вҶ’ click "йҖүжӢ©жӯӨйўҳзӣ®" вҶ’ `extractSamples()` extracts `<pre>` blocks paired by "ж ·дҫӢиҫ“е…Ҙ"/"ж ·дҫӢиҫ“еҮә" headings вҶ’ `writeSamplesToCache()` writes `testN.in`/`testN.out` to `QStandardPaths::TempLocation + "/SM-OJ-<timestamp>"` вҶ’ emit `sampleSelected(folderPath)` вҶ’ `MainWindow::onOpenJudgeSampleSelected()` вҶ’ `JudgePanel::setTestFolder(path)` вҶ’ judge ready.
- **OpenJudge submission**: `JudgePanel "жҸҗдәӨеҲ°OpenJudge" button` вҶ’ `emit submitToOpenJudgeRequested()` вҶ’ `MainWindow::onSubmitToOpenJudge()` (checks: code file вҶ’ problem selected via `hasCurrentProblem()` вҶ’ login вҶ’ maps extension вҶ’ langId: .cвҶ’0, .cpp/.cc/.cxxвҶ’1, .py/.pywвҶ’6) вҶ’ `OpenJudgeWindow::submitCurrentProblem(code, langId)` (checks problem selected вҶ’ login вҶ’ ongoing homework) вҶ’ `Crawler::submitCode()` вҶ’ GET `/hwXXXXX/A/submit/` вҶ’ extract contestId, problemNumber, language radio вҶ’ POST to `/api/solution/submitv2/` (raw source, percent-encoded, no base64) вҶ’ parse JSON `redirect` URL вҶ’ `doPollSubmissionStatus()` via QTimer (2s interval, max 15 attempts = 30s timeout) вҶ’ parse solution page text (`зҠ¶жҖҒ: Accepted`, `ж—¶й—ҙ: 23ms`, `еҶ…еӯҳ: 7272kB`) вҶ’ emit `submissionResultReady()` вҶ’ `MainWindow::onSubmissionResultReady()` вҶ’ show `SubmitResultPanel` in right splitter (1/3 height), hide OutputPanel.

### Component Details

**EditorWidget** вҖ” Four-mode editor via `QStackedWidget`: page 0 = `WikiLinkTextEdit` (Markdown/generic text), page 1 = `QWebEngineView` container (full-screen Markdown preview), page 2 = `CodeEditor` (code files), page 3 = `QSplitter(Qt::Horizontal)` with `m_textEdit` (left) + second `QWebEngineView` (right) for split-pane preview. Mode auto-selected in `loadFile()` via `LanguageUtils::languageForExtension()`. Code mode skips preview toggles (`setPreviewMode`/`setSplitPreviewMode` are no-ops). Full preview and split preview are mutually exclusive; the `m_textEdit` widget is transferred between page 0 and page 3's splitter via `removeWidget()`/`addWidget()` when entering/exiting split mode. Split preview uses a 500ms debounce timer (`m_splitDebounceTimer`) that only refreshes when content differs from `m_lastSplitPreviewContent`. Both preview modes share the same `preparePreviewContent()` pipeline, `PreviewPage` navigation interception, and zoom synchronization (`applyZoom()` sets zoom on all active WebEngine views). Preview uses an HTML template that loads marked.js for MarkdownвҶ’HTML conversion, KaTeX for LaTeX math rendering, and Mermaid.js for diagram rendering. WikiLink detection happens during preview HTML generation via regex (`[[...]]` вҶ’ `<a href="wikilink:...">`), intercepted by a custom `QWebEnginePage::acceptNavigationRequest()` override. Code blocks in preview are pre-highlighted via C++ (`preparePreviewContent` вҶ’ `preHighlightCodeBlocks` вҶ’ `highlightCodeBlock`: direct regex-based highlighting using ConfigManager colors, matching CppSyntaxHighlighter/PythonSyntaxHighlighter rules), then base64-encoded into `highlighted` custom fenced blocks decoded by the marked.js renderer. **Preview code block run**: marked renderer adds в–¶ Run button on fenced code blocks (`python`/`cpp`/`c`/`py`/`cc`/`cxx`); click stores code in JS `window._runCodeData` вҶ’ navigates to `runblock:execute` вҶ’ `acceptNavigationRequest()` intercepts `runblock:` scheme вҶ’ `runJavaScript` reads stored data вҶ’ emits `runCodeBlockRequested` signal (forwarded from `EditorWidget`). Zoom factor range: 0.5вҖ“3.0, step 0.1. Has a 300ms debounce timer that auto-clears the modified flag if text reverts to original content. All content access methods (`toPlainText()`, `setPlainText()`, `isModified()`, `setModified()`, `applyZoom()`, `scrollToLine()`, `clearExtraSelections()`) branch on `m_editorMode` enum (`MarkdownEdit` vs `CodeEdit`).

**FileExplorerWidget** вҖ” Uses `QFileSystemModel` + `QSortFilterProxyModel` (custom `FileSortProxyModel` for folders-first sort). Custom `NoGhostDelegate` eliminates text ghosting during inline rename; if rename results in empty name, auto-restores original. Supports drag-drop file moving (within root directory only) with visual feedback (blue indicator bar). Delete key triggers removal even from context menu via `DeleteKeyFilter` event filter. Has a breadcrumb path bar at the top using `FlowLayout` for auto-wrapping: clickable directory segments (dimmed ancestors, white current) + `>` separators. `updateBreadcrumb()` rebuilds on `setRootPath()`. `selectFile(filePath)` syncs tree selection to active tab by mapping sourceвҶ’proxy index, expanding parent folders, and calling `setCurrentIndex()` + `scrollTo()`.

**TabManager** вҖ” Custom `CustomTabBar` (extends QTabBar) constrains drag bounds. `closeTab` shows a custom save prompt dialog. `closeAllTabs` is used for graceful shutdown. `updatePathsAfterMove` handles batch path updates after file tree drag-move operations.

**HistoryPanel** вҖ” Max 50 entries, auto-dedup by path. Persisted via `SettingsManager` under `History/recentFiles`. Persistence is deferred to program shutdown only (via `MainWindow::closeEvent`) to reduce disk I/O. `replacePath` rebuilds UI list after file moves. Automatically removes entries for deleted files on next access (shows warning dialog).

**SearchPanel** вҖ” Full-text search panel in a left-side `QDockWidget` (toggle: `Ctrl+Shift+F`). Contains `QLineEdit` for keyword input (300ms debounce), `QListWidget` for results (filename + line number + context snippet), and a `QLabel` status bar. On search, uses `QDirIterator` + `TextFileUtils::scanNameFilters()` to recursively scan all text files, reads each file line-by-line via `QTextStream`, and performs case-insensitive `QString::contains` matching. Results are capped at 20 matches per file and 500 total. On result click, emits `resultClicked(filePath, lineNumber, searchText)` вҶ’ `MainWindow` opens file and calls `EditorWidget::scrollToLine`, which highlights all matching occurrences with gold (`#FFD700`) background via `QTextEdit::setExtraSelections`. Unlike History/Backlinks panels, SearchPanel does NOT auto-hide on outside click (persistent sidebar behavior).

**BacklinkIndex** вҖ” Reverse index: absolute path of target file вҶ’ list of source file paths that link to it via `[[wikilinks]]`. Also maintains a forward index (source вҶ’ targets) for incremental updates. Uses the same recursive regex `\[\[((?:[^\[\]]|\[(?1)\])*)\]\]` as `EditorWidget::refreshPreview`. WikiLink target resolution follows deterministic rules (exact path under root for any known text extension вҶ’ index lookup by `completeBaseName` вҶ’ shortest-path tiebreaker), unlike `findWikiTarget` which uses current-editor-context disambiguation. Uses `TextFileUtils::scanNameFilters()` for file scanning and `TextFileUtils::textExtensions()` for target resolution. Supports async full scan via `BacklinkData` struct + `static buildFromPath()` (runs on worker thread, returns self-contained data) + `setData()` (main-thread swap via `std::move`). `resolveTarget` is static so it can be called from any thread. Incremental ops (`rebuildFile`, `onFileRenamed`, `onFileDeleted`) remain synchronous. `onFileRenamed` migrates both backlink keys and forward-link target references to keep `rebuildFile вҶ’ removeFile` cleanup consistent.

**BacklinksPanel** вҖ” Display-only `QListWidget` (NoSelection) showing source files that link to the currently open file. Follows HistoryPanel pattern: embedded in `QDockWidget` (right side, default hidden), toolbar toggle (Ctrl+Shift+B), auto-hide on outside click. Shows "ж— еҸҚеҗ‘й“ҫжҺҘ" placeholder when empty. Minimum width 200px to prevent panel collapse.

**TagIndex** вҖ” Plain C++ class (non-QObject). Bidirectional index: `m_tagToFiles` (tag вҶ’ file paths) and `m_fileToTags` (file вҶ’ tags for incremental cleanup). `buildFromPath()` scans only `*.md`/`*.markdown` files. `extractTagsFromContent()` uses regex `(*UCP)#(?!\s)([\w-]+)` вҖ” Unicode-aware, excludes `# Heading`, skips ``` code blocks. `processTagsForPreview()` converts `#tag` вҶ’ `<a href="tag:tag">#tag</a>` for preview rendering. `setData()` swaps data atomically for async build. Incremental ops: `rebuildFile`, `onFileRenamed`, `onFileDeleted`. Async build is Phase 3 in `startAsyncIndexBuild()` (after file index and backlinks).

**TagPanel** вҖ” Dual-mode `QListWidget` panel: `showAllTags()` displays all tags with `#` prefix; `showFilesForTag()` lists files containing a specific tag. Back button returns to tag list. `m_showingFiles` flag controls click dispatch (emit `tagClicked` vs `fileClicked`). Empty states in gray. Follows History/Backlinks dock pattern: 200px min width, `Ctrl+Shift+T` toggle, auto-hide on outside click.

**ConfigManager** вҖ” Singleton loaded before MainWindow construction. Reads `config.json` (project root or executable dir) with dot-path resolution (`"editor.zoom.min"`). All accessors return built-in defaults if the file is missing. Covers colors, fonts, editor/panel sizes, judge limits, compiler flags, OpenJudge URLs/timeouts, shortcuts, extensions list, etc.

**SettingsManager** вҖ” Writes `config.ini` next to the executable. Stores runtime session state only: window geometry, splitter state, last open/save-as folder paths, recent files list, and OJ credentials (base64-obfuscated). Coexists with ConfigManager without overlap.

**TextFileUtils** (`fileutils.h`) вҖ” Header-only utility providing `textExtensions()` (returns 40+ common text file extensions like `md`, `txt`, `cpp`, `py`, `js`, `json`, `xml`, `yaml`, `csv`, etc.) and `scanNameFilters()` (returns `"*." + ext` filters for `QDirIterator`). Used by `BacklinkIndex`, `EditorWidget`, `FileExplorerWidget`, and `MainWindow` to replace hardcoded `.md`/`.txt` file lists.

**WikiLinkTextEdit** (`wikilinktextedit.h/cpp`) вҖ” `QTextEdit` subclass with built-in `QCompleter` + `QStringListModel` for `[[wikilink]]` autocomplete. On `[[` detection via `QTextBlock::text()` + `lastIndexOf`, shows a popup listing filenames from `m_fileIndex` (no extension, case-insensitive prefix match). Also supports `#tag` autocomplete: `findHashTag()` helper scans backward from cursor for `#` at word boundary, switches completer model to `m_tagModel`, Tab inserts tag name. First item auto-selected; arrow keys navigate, Tab accepts and inserts `[[filename]]` with cursor after `]]`. `QCompleter::popup()` positioned at `cursorRect()`. Model populated via `EditorWidget::setFileNames` вҶ’ `WikiLinkTextEdit::setFileNames`, called from `MainWindow::updateCurrentEditorCompletions` after index rebuilds and tab switches.

**CodeEditor** (`codeeditor.h/cpp`) вҖ” `QPlainTextEdit` subclass for code editing. Features: `LineNumberArea` inner `QWidget` for line numbers (`#858585` on `#252525`); auto-indent on Enter (copies leading whitespace, adds level on `{`, splits `{|}` to 3 lines); bracket completion for `{}()[]""''` (wraps selection, skips in string/comment); closing bracket skip-over; backspace pair removal (deletes empty pair); backspace indent deletion (removes `m_indentWidth` spaces at a time in leading whitespace); Tab inserts `indentString()` (4 spaces) or indents selection; current line highlight (`#2A2D2E`); search highlight storage (`m_searchHighlights`, merged with current line in `highlightCurrentLine()` via `setExtraSelections`). Constructor sets dark stylesheet (`#1E1E1E` background), Consolas 12pt monospace font, no-wrap mode. `setLanguage(langId)` installs/replaces `QSyntaxHighlighter` via `LanguageUtils::createHighlighter()`.

**LanguageUtils** (`languageutils.h/cpp`) вҖ” Extensible language registry. `LanguageInfo` struct holds display name, extension set, and highlighter factory function. Singleton `languageMap()` returns static `QMap<langId, LanguageInfo>`. Key functions: `languageForExtension(ext)` looks up language by file extension; `isCodeFile(ext)` returns whether extension is registered; `createHighlighter(langId, doc)` invokes the factory; `normalizeCodeFenceLanguage(fenceLang)` maps markdown code fence language strings (e.g. `py`/`c`) to canonical keys (`python`/`cpp`). Currently registers: `"cpp"` (extensions: `cpp, hpp, cxx, cc, c, h, hxx, hh`) вҶ’ `CppSyntaxHighlighter`; `"python"` (extensions: `py, pyw, pyx`) вҶ’ `PythonSyntaxHighlighter`. Adding a language = 1 map entry + 1 highlighter file + `.pro` entries.

**CppSyntaxHighlighter** (`cppsyntaxhighlighter.h/cpp`) вҖ” `QSyntaxHighlighter` for C/C++. Dark theme colors: keywords `#569CD6` (bold, includes C++20), preprocessor `#C586C0`, types `#4EC9B0`, strings `#CE9178`, numbers `#B5CEA8`, comments `#6A9955` (single-line `//` and multi-line `/* */` with block-state tracking). Rules stored in `QVector<HighlightingRule>`, applied in priority order.

**PythonSyntaxHighlighter** (`pythonsyntaxhighlighter.h/cpp`) вҖ” `QSyntaxHighlighter` for Python. Dark theme colors: keywords `#569CD6` bold (`def`, `class`, `if`, `for`, `import`, etc.), builtin types/functions `#4EC9B0` (`int`, `str`, `list`, `print`, etc.), constants `#569CD6` bold (`True`, `False`, `None`), decorators `#C586C0` (`@...`), `self`/`cls` `#DCDCAA`, strings `#CE9178` (including f-strings and raw strings), numbers `#B5CEA8`, comments `#6A9955`. Triple-quoted strings (`"""`/`'''`) tracked across blocks via `setCurrentBlockState()`/`previousBlockState()`.

**CompilerUtils** (`compilerutils.h`) вҖ” Header-only utility. `findCompilers()` detects g++ (via `QStandardPaths::findExecutable`) and MSVC cl.exe (when `VSCMD_VER` env var is set). `findPython()` detects `python` or `python3`. `getCompileArgs(id, src, out)` returns `g++ -std=c++17 -Wall -Wextra` or `cl /std:c++17 /W4 /EHsc` flags. `getOutputPath(src)` derives the `.exe` path from the source file.

**ProcessRunner** (`processrunner.h/cpp`) вҖ” `QObject` managing a two-phase compileвҶ’run pipeline via sequential `QProcess` instances. `startCompile(source)` compiles via `CompilerUtils::defaultCompiler()`; `startRun(executable)` runs the binary; `startCompileAndRun(source)` does both, auto-transitioning to run on compile success; `startRunPython(source)` invokes the python interpreter on a `.py` file. `stop()` kills the current process. `writeInput(text)` writes `text + "\n"` to the running process's stdin; `writeRaw(text)` writes without appending `\n`. `isAcceptingInput()` returns true only in `RunOnly` mode (prevents input during compilation). Output emitted via `outputReceived(text, isStderr)` вҖ” no `.trimmed()` applied, raw format preserved. Signals: `compileFinished(success)`, `runFinished(exitCode)`, `processStarted()`, `processStopped()`.

**JudgeEngine** (`judgeengine.h/cpp`) вҖ” `QObject` managing the local judge pipeline. `discoverTests()` scans folder for matching `.in`/`.out` pairs, skipping empty `.out` files. `start()` compiles via `CompilerUtils` spawns `QProcess`, then runs a warmup execution (empty stdin, discarded output) to populate OS cache before the first real test case. Each test runs the child `QProcess`, feeds `.in` content to stdin, captures stdout, and compares line-by-line (trimmed, trailing blank lines stripped). Timeout: 1000ms via single-shot `QTimer` (TLE). Memory monitoring uses triple-capture strategy: (1) synchronous `captureMemory()` right after `process.start()` while the child blocks on stdin; (2) 100ms `QTimer` polling via shared `captureMemory()` with `PROCESS_QUERY_LIMITED_INFORMATION`; (3) one final read on process exit. Peak > 65536KB triggers MLE and kills the process. `m_testHandled` flag prevents dual-fire between timeout and process finish. Emits `compileFinished`, `testStarted`, `testFinished`, `allTestsFinished`.

### Naming Convention

- `[[ж–Үд»¶еҗҚ]]` (WikiLink syntax) вҖ” links by filename without path or extension
- `findWikiTarget` performs: exact match вҶ’ parent-dir-first search вҶ’ sibling-subtree search вҶ’ full subtree search
- `BacklinkIndex::resolveTarget` performs: exact path under root вҶ’ global index lookup by `completeBaseName` вҶ’ shortest-path tiebreaker (no current-editor bias)

## Coding Standards
- **Qt Logic**: Always use the **new signal/slot syntax**: `connect(sender, &Sender::signal, receiver, &Receiver::slot)`.
- **I18n**: Wrap all user-visible strings in `tr()` for future translation support (e.g., `tr("Save File")`).
- **Naming**: 
  - Private members should start with `m_` (e.g., `m_fileIndex`).
  - Use `camelCase` for methods and variables, `PascalCase` for classes.
- **Memory Management**: Rely on Qt's **parent-child system** for automatic memory cleanup. If a class is not a `QObject`, use `std::unique_ptr` where appropriate.
- **Header Guards**: Use `#ifndef FILENAME_H` style guards instead of `#pragma once`.

## UI Guidelines
- **Layouts**: Primarily use `QSplitter` for the main workspace and `QLayout` subclasses for toolbars/status bars.
- **Theming**: Consolidate visual styling in `setStyleSheet` calls. New widgets should match the dark-themed tab styles defined in `TabManager`.
- **Panel Behavior**: Side panels (History, Backlinks) must be hosted in `QDockWidget` and implement the **"click-outside-to-hide"** pattern via `MainWindow::eventFilter`.
- **Tree View**: Any changes to the File Explorer must respect the `NoGhostDelegate` to prevent text rendering artifacts during renaming.
